Project write-up

MicroFit Coach capstone blog and deployment summary.

This write-up documents the purpose, cloud services, visitor counter design, IaC plan, CI/CD pipeline, monitoring, and lessons learned.

Project purpose

The goal is to build a fitness coaching landing page that feels like a real client dashboard while demonstrating cloud architecture and deployment readiness. It supports macro tracking, workout logging, coach contact, visitor counting, team pages, and documentation for a 4-week capstone.

Cloud services used

  • Static hosting with AWS S3 and CloudFront (or Azure Blob Storage + CDN)
  • Python API implemented as an AWS Lambda / Azure Function
  • NoSQL counter table with DynamoDB / Cosmos DB
  • Monitoring with AWS CloudWatch / Azure Monitor

Visitor counter architecture

The website calls a serverless API endpoint that reads and increments a visitor counter in a NoSQL database. The API returns the updated count to display on the landing page, and the front end gracefully falls back when the cloud endpoint is unavailable.

Infrastructure as Code

The project includes Terraform templates to provision static hosting, serverless functions, the database table, IAM roles, and optional CDN resources. IaC keeps deployments repeatable and version controlled.

CI/CD pipeline

A GitHub Actions workflow validates Terraform, builds the site, and deploys static assets to the cloud. Secrets are stored in GitHub Secrets, and the pipeline can trigger on main branch pushes.

Monitoring and security

Use cloud monitoring to track API requests, function errors, and visitor count updates. Configure HTTPS with a custom domain and use least-privilege IAM roles for the Lambda function and S3 access.

Challenges and lessons learned

  • Balancing a polished landing page with a simple demo-ready backend.
  • Structuring the app for future database-connected macros and workouts.
  • Keeping deployment infrastructure readable for team review and grading.
